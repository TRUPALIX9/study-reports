Task 3: Threat Tree Diagram (TTD) (3 Points)

Root Threat

"Compromise of WordPress Website Integrity and User Data"

Vulnerabilities

  • Weak Authentication ‚Äì Weak passwords or missing 2FA allow brute-force attacks.
  • SQL Injection ‚Äì Improperly sanitized input in plugins/themes.
  • Cross-Site Scripting (XSS) ‚Äì User input not properly escaped in pages or comments.

Countermeasures

  • Strong Password Policy & 2FA ‚Äì Enforce complex passwords and MFA.
  • Parameterized Queries & ORM Use ‚Äì Prevent SQLi by avoiding dynamic queries.
  • Input Sanitization & CSP Headers ‚Äì Prevent XSS and injected script execution.

Task 4: Use & Misuse Diagram (UMD) (3 Points)

Use Cases (Legitimate)

  • User Logs Into Account ‚Äì Authenticates through the login form.
  • Admin Publishes a Post ‚Äì Uses dashboard to upload and edit content.
  • User Comments on a Post ‚Äì Interacts with front-end commenting feature.

Misuse Cases (Attacks)

  • Credential Stuffing Attack ‚Äì Attacker attempts login with stolen credentials.
  • Malicious File Upload ‚Äì Attacker uploads a malicious plugin or media file.
  • Comment XSS Injection ‚Äì Attacker posts a script in a comment field.

Mitigations

  • Rate Limiting & MFA ‚Üí Protects against Credential Stuffing.
  • File Type Validation & Security Plugins ‚Üí Stops Malicious Uploads.
  • Input Sanitization & Output Encoding ‚Üí Prevents XSS Injection.
